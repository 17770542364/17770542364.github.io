推流中断与画质不清晰的解决办法

推流中断的原因：音频和视频播放不同步，时间戳出现错误

解决：加入参数 -re

画质不清晰的原因：播放时按照默认设置对视频重新编码，画质有所下降

解决：加入参数 -c copy

Linux下安装ffmpeg

下载ffmpeg

http://ffmpeg.org/download.html

得到ffmpeg源代码并解压缩

tar -xjf ffmpeg-snapshot.tar.bz2

对于以.tar.bz2格式为后缀的压缩文件需要添加 -j 参数进行解压

编译ffmpeg

# --prefix指定安装的位置 --enable-debug 打开debug选项 
# --disable-static --enable-shared 关闭生成静态库，打开生成动态库
./configure --prefix=/usr/local/ffmpeg --enable-debug=3 --disable-static --enable-shared
# 指定四个核并发编译
make -j 4
sudo make install

./configure时出现提示

nasm/yasm not found or too old. Use –disable-x86asm for a crippled build.

安装yasm包

WARNING: pkg-config not found, library detection may fail.

安装pkg-config包

编译好之后/usr/local/ffmpeg下的目录:

编译之后没有ffplay

fplay编译主要依赖sdl2环境(针对ffmpeg 3.x版本)，未安装sdl2的情况下，不会自动生成关于ffplay的编译选项。

SDL(Simple DirectMedia Layer)是一个跨平台的多媒体和游戏开发包，提供2D，音频，事件驱动，多线程和定时器等服务，它使用C语言写成，提供了多种控制图像、声音、输出的函数，让开发者只要用相同或是相似的代码就可以开发出跨多个平台（Linux、Windows、Mac OS X等）的应用软件。

可以通过下面两个命令安装sdl2组件，dev后缀表示安装路径下会包含头文件及动态库等在开发环境下需要的文件

sudo apt-get install libsdl2-2.0

sudo apt-get install libsdl2-dev

重新编译

ffmpeg添加libfdk_aac

1.安装fdk_aac

wget https://sourceforge.net/projects/opencore-amr/files/fdk-aac/fdk-aac-2.0.2.tar.gz
tar -zxvf fdk-aac-2.0.2.tar.gz
cd fdk-aac-2.0.2/
./autogen.sh
# 如果出现错误Can't exec "libtoolize": 没有那个文件或目录 at /usr/share/autoconf/Autom4te/FileUtils.pm line 345, <GEN3> line 5.
# 安装libtool sudo apt install libtool
./configure CFLAGS=-fPIC
make -j
sudo make install

重新编译ffmpeg，添加libfdk_acc编译选项

cd ffmpeg
./configure --prefix=/usr/local/ffmpeg --enable-debug=3 --disable-static --enable-shared --enable-libopus --enable-encoder=libopus --enable-libfdk-aac
make -j 4
sudo make install

将pkg路径添加到PKG_CONFIG_PATH中去

vim ~/.bashrc

# 添加以下内容
export PKG_CONFIG_PATH="/usr/local/lib/pkgconfig:$PKG_CONFIG_PATH"

ffmpeg安装OPUS

安装opus

sudo apt-get install libopus-dev

重新编译ffmpeg，添加libopus编译选项

cd ffmpeg
./configure --prefix=/usr/local/ffmpeg --enable-debug=3 --disable-static --enable-shared --enable-libopus --enable-encoder=libopus --enable-libfdk-aac
make -j 4
sudo make install

ffmpeg安装x264库

# 安装x264库指令
sudo apt-get install libx264-dev

后续步骤与安装opus库步骤相同, 编译时加入参数--enable-libfdk-aac

配置环境变量

添加PATH路径和PKG_CONFIG_PATH路径

vim ~/.bashrc

添加ffmpeg的bin路径

export PATH="/usr/local/ffmpeg/bin:$PATH"

export PKG_CONFIG_PATH="/usr/local/ffmpeg/lib/pkgconfig:$PKG_CONFIG_PATH"

重新加载文件

source ~/.bashrc

将ffmpeg库文件地址加入到默认搜索路径

sudo vim /etc/ld.so.conf

在文件中加入路径

/usr/local/ffmpeg/lib

更新环境变量

sudo ldconfig

配置了~/.bashrc但每次登陆后需要重新source的问题

这个问题就在于，系统没有自动的执行 .bashrc 这个文件，导致我们每次登陆时候都需要source 这个文件。

如果不source 这个文件最大的影响就是文件夹和文件我们看不出来，都是白色的。source 之后呢，文件夹就会变成蓝色的，可执行文件就会变成绿色的，这样更方便我们工作。

Linux启动时读取配置文件的顺序

在刚登录Linux时，首先启动/etc/profile 文件，然后再启动用户目录下的 ~/.bash_profile、~/.bash_login或 ~/.profile文件中的其中一个。
具体执行的顺序为：~/.bash_profile、~/.bash_login、 ~/.profile。

如果 ~/.bash_profile文件存在的话，一般还会执行 ~/.bashrc文件。
因为在~/.bash_profile文件中会有以下内容：

# .bash_profile
        
if [ -f ~/.bashrc ]; then
    . ~/.bashrc       
fi

~/.bashrc中一般有以下内容，这是执行/etc/下总的bashrc 文件，全局环境变量。

# .bashrc

# Source global definitions
if [ -f /etc/bashrc ]; then
 		. /etc/bashrc
fi

综上，执行顺序可以总结为：
/etc/profile→ (~/.bash_profile | ~/.bash_login | ~/.profile)→~/.bashrc →/etc/bashrc

所以，要自动的执行~/.bashrc ，只需要将以下代码，放到~/.bash_profile文件里就好了，如果没有这个文件就新建一个。

# .bash_profile
        
if [ -f ~/.bashrc ]; then
    . ~/.bashrc                                
fi
